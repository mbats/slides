We expect that they understand that these modules are connected to a platform which contains software.

Open hardware

Nowadays, it is really easy to get good material to learn electronics thanks to the open hardware initiative. What is open hardware ? Open source hardware is a licensing agreement for electromechanical projects. That is to say, electronics and robotics and other mechanical projects are open hardware if they are documented and published under an open hardware license.

Electronic for kids

Open hardware dedicated to kids learning electronics : littleBits

The problem with electronic is that it is most of the time small things. So it is not so easy for kids to manipulate. And it is so easy to do a bad connection and just damage your electronic module. An open hardware company proposed really cool electronic modules to solve this issue. littleBits is an opensource library of electronic modules that snap together with tiny magnets. There is no soldering, no wiring, no programming. It's great to understand electronics but not to understand programming and to perceive and comprehend the connection between software and hardware modules.

Arduino

The most famous open hardware platform is Arduino. With Arduino you get a piece of hardware, but also documentation and a development environment that fits all together with hardware. The arduino board is like a small computer that can be programmed as many times as needed. As a computer, it provides I/O interaction, through digital and analog pins. Arduino can sense the environment by receiving input from a variety of sensors and can affect its surroundings by controlling lights, motors, and other actuators. The microcontroller on the board is an Atmel AVR microcontroller and usually is programmed using the Arduino programming language and the Arduino development environment.

Understand programming

Many open source software are dedicated to learn programming to kids : Scratch, TurtleLogo, KidsRuby, Python4Kids...

Initiatives : Programatoo, Devoxx4Kids, greenlight, coding goûter...

To understand the world the kids have also to learn programming. Many different initiatives are arising to learn programming to kids. Theses movements use many different kind of software : open source software or proprietary software. The most famous, TurtleLogo is for the eldest ones. More recently, Scratch has become more popular. These tools have different approaches.

Text

In TurtleLogo kids write text to move a small turtle cursor in order to draw shapes. No graphical UI, very simple language. They learn the concepts of programming and understand that a program is just a suite of instructions.

Graphical

In Scratch you drag and drop elements from a palette to animate a character. This tool is a complete IDE which allows kids to create cartoons, video games, movies... This tool provides a more graphical UI. Kids have to assemble puzzle pieces which are instructions to describe a program.

Program the real world

These tools are great to learn what programming is. It exists other kind of tools that allows kid to program the real world by interacting with sensors and actuators. Open source tools like Scratch4Arduino or Ardublock provide UIs for simple programming of the Arduino platform. Thanks to these tools, kids can develop a software, upload and run it on hardware platforms.

Turning Eclipse into an IDE for kids

Graphical programming & light UI

The idea was to develop a prototype to allow kids to create small program. Our first actions were to simplify the UI, the project lifecycle and to provide a graphical block language to be able to develop simple programs and run them on electronic modules. The purpose is not to provide a complete IDE but a tool giving the possibility to discover the effects a software program could have on electronic modules.

A simplified UI

No menu, only a toolbar with 4 buttons activated according to what is possible

The UI of the Arduino designer defines no menu just one toolbar with 4 different buttons to : manage the project, define the hardware modules used and to define the software that would be uploaded on the hardware platform.

Dashboard

    The process :
  1. Define connections between hardware platform and modules
  2. Describe the sketch
  3. Upload on the target
A dashboard helps the kid by indicating what is the next step in the process development. The process is really simple in fact. The kid must describe the connection between the hardware modules. Then he describes the software and finally he uploads it to the target.

Hardware

Platform, Modules, Wire

First step, the kid defines the hardware platform he is using and the different sensors and actuators he connects to the Arduino.

Sketch

Software written using Arduino are called sketches. Then the kid describes the sketch which represents the software that would run on the hardware platform. And finally, he uploads the sketch to the target.

Graphical block-based language

  • Structures : while, repeat, if
  • Functions : delay
  • Hardware modules : Status, Level, Sensor
  • Variables
  • Constants
  • Mathematical operators
As you can see in the Arduino designer palette, I defined a simple graphical block based language. With this language its possible to create instructions like while, repeat or if instructions. Then it is possible to use functions like delay and to reference hardware modules. It is also possible to create variables and constants and some mathematical operators. For this prototype I decided to create a low level language which is still close to the Arduino language but we could imagine to provide higher level functions.

Sirius

A tool to quickly define DSL based custom multi-view workbenches with dedicated representations

To develop the graphical designers I used Sirius. Sirius is a new project in Eclipse to quickly define multi view workbenches based on DSL with dedicated representations.

Why using Sirius ?

  • For the tool development phase : Dynamic & iterative (live) development
  • For kids : Tooling adapted to the development of small Arduino projects
So why am I using Sirius? During the prototype development phase, it allows me to develop really quickly an arduino specific modeler. The development is easy, dynamic and iterative thanks to a live preview. I choose it also because I am used to work with Sirius in my daily work, and I just want to develop with Sirius something cool. For the kids, they get a tooling completely adapted to the development of small arduino projects and simple to use.

Step 3 - Code generator

Generate ino files with Acceleo 


int brightness;
void setup() {
  brightness=0;
}
void loop() {
 while ((brightness<255))
 {
  brightness=(brightness+1);
  analogWrite(11,255-(brightness));
  delay(8);
 }
 while ((brightness>1))
 {
  brightness=(brightness-1);
  analogWrite(11,255-(brightness));
  delay(8);
 }
 delay(200);
}
Next step was to develop a small generator with Acceleo to generate from Arduino's model the sketch. In Arduino, sketches are saved with the file extension .ino. The Arduino language is based on C/C++. It links against AVR Libc and allows the use of any of its functions. The AVR Libc package provides a subset of the standard C library for Atmel AVR microcontrollers like Arduino. If we have a look to the generated code : we can see that Arduino programs can be divided in three main parts: structure (setup, loop, if, repeat, while...), values (variables and constants), and functions (pinMode, digitalWrite, analogRead...). The setup() function is called when a sketch starts. It is used to initialize variables, pin modes, start some libraries, etc. The setup function will only run once, after each powerup or reset of the Arduino board. After generating the setup() function, my generator create a loop() function. This loops consecutively, allowing the program to change and respond.

Step 4 - Integrate Arduino compiler and target uploader

Use Arduino IDE to get well packaged tools

avr-gcc, avrdude

When you use the Arduino environment the compile and upload actions are directly integrated. Here as I want to integrate the generation and the compilation of ino files in my Eclipse application, I searched how to compile and upload Arduino programs from the command line. To be able to upload the code on the target, I also installed avr-gcc which is a cross-compiler for the AVR chip that runs the Arduino and avrdude. avrdude is a very handy utility for loading code onto AVR microcontrollers. As Arduino ships with its own bootloader, it is possible to load and run code using only a single USB cable connecting the computer to the board. Uploading the code on the hardware platform is then really easy. My Eclipse upload action has just to call the avrdude command.
