Open hardware

Arduino

Nowadays, it is really easy to get good material to learn electronics thanks to the open hardware initiative. What is open hardware ? Open source hardware is a licensing agreement for electromechanical projects. That is to say, electronics and robotics and other mechanical projects are open hardware if they are documented and published under an open hardware license. The most famous open hardware platform is Arduino. With Arduino you get a piece of hardware, but also documentation and a development environment that fits all together with hardware.

Arduino kits

Cheap, easy to find thanks to many different providers : AdaFruit, DFRobot, SeeedStudio, SnootLab, ...

So many kinds of modules available!

What's great with arduino is that it is cheap and really easy to find. There are many efficient providers available on internet like adafruit, DFRobot, SeeedStudio. They provide an incredible amount of modules! Order and two days later you will receive at home your new toy.

Create with electronic

Open hardware dedicated to kids learning electronics : LittleBits

The problem with electronic is that it is most of the time small things. So it is not so easy for kids to manipulate. And it is so easy to do a bad connection and just damage your electronic module. Another open hardware company proposed really cool electronic modules to solve this issue : littleBits. littleBits is an opensource library of electronic modules that snap together with tiny magnets. There is no soldering, no wiring, no programming. It's great to understand electronics but not to understand programming and to perceive and comprehend the connection between software and hardware modules.

Understand programming

Many open source software are dedicated to learn programming to kids : Scratch, TurtleLogo, KidsRuby, Python4Kids...

Initiatives : Programatoo, Devoxx4Kids, greenlight, coding goûter...

To understand the world the kids have to learn programming. Many different initiatives are arising to learn programming to kids. Theses movements use many different kind of software : open source software, proprietary software, online software. The most famous, TurtleLogo is for the eldest ones. More recently, Scratch has become more popular. These tools have different approaches.

Text

In TurtleLogo kids write text to move a small turtle cursor in order to draw shapes. No graphical UI, very simple language. They learn the concepts of instructions and understand that a program is just a suite of instructions.

Graphical

In Scratch you drag and drop elements from a palette to animate a character. This tool is a complete IDE which allows kids to create cartoons, video games, movies... This tool provides a more graphical UI. Kids have to assemble puzzle pieces to describe instructions.

Program the real world

These tools are great to learn what programming is. It exists other kind of tools that allows kid to program the real world by interacting with sensors and actuators. Open source tools like Scratch4Arduino or Ardublock provide UIs for simple programming of the Arduino platform. Thanks to these tools, kids can develop a software, upload and run it on hardware platforms.

Turning Eclipse an IDE for kids

Graphical programming & light UI

The idea was to develop a prototype to allow kids to create small program. Our first actions were to simplify the UI, the project lifecycle and to provide a graphical block language to be able to develop simple programs and run them on electronic modules. The purpose is not to provide a complete IDE but a tool giving the possibility to discover the effects a software program could have on electronic modules.

A simplified UI

No menu, only a toolbar with 4 buttons activated according to what is possible

The UI defines no menu just one toolbar with 4 different buttons to : manage project lifecycle, define the hardware modules used and to define the sketch that would be uploaded on the hardware platform.

Dashboard

    Indicate the process :
  • Define connection between hardware platform and modules
  • Describe the sketch
  • Upload on the target
A dashboard helps the user by indicating what is the next step in the process development. The process is really simple in fact. The user must describe the connection between the hardware modules. Then he describes a sketch and finally he uploads the sketch to the target.

Hardware

Platform, Modules, Wire

First step, the kid defines the hardware platform he is using and the different sensors and actuators he connects to the Arduino.

Sketch

Graphical block-based language

Software written using Arduino are called sketches. Then the kid describes the sketch which represents the software that would run on the hardware platform. And finally, he uploads the sketch to the target.

Sirius

A tool to quickly define DSL based custom multi-view workbenches with dedicated representations

To develop the graphical designers I used Sirius. Sirius is a new project in Eclipse to quickly define multi view workbenches based on DSL with dedicated representations.

Why using Sirius ?

  • For the tool development phase : Dynamic & iterative (live) development
  • For kids : Tooling adapted to the development of arduino small projects
So why am I using Sirius? During the prototype development phase, it allows me to develop really quickly an arduino specific modeler. The development is easy, dynamic and iterative thanks to a live preview. I choose it also because I am used to work with Sirius in my daily work as I use it also to develop another software UML Designer. For the kids, they get a tooling completely adapted to the development of small arduino projects and really simple to use.

Step 3 - Code generator

Generate ino files with Acceleo 


int brightness;
void setup() {
  brightness=0;
}
void loop() {
 while ((brightness<255))
 {
  brightness=(brightness+1);
  analogWrite(11,255-(brightness));
  delay(8);
 }
 while ((brightness>1))
 {
  brightness=(brightness-1);
  analogWrite(11,255-(brightness));
  delay(8);
 }
 delay(200);
}
Next step was to develop a small generator with Acceleo to generate from Arduino's model the sketch. In Arduino, sketches are saved with the file extension .ino. The Arduino language is based on C/C++. It links against AVR Libc and allows the use of any of its functions. The AVR Libc package provides a subset of the standard C library for Atmel AVR microcontrollers. If we have a look to the generated code : we can see that Arduino programs can be divided in three main parts: structure (setup, loop, if, repeat, while...), values (variables and constants), and functions (pinMode, digitalWrite, analogRead...). The setup() function is called when a sketch starts. It is used to initialize variables, pin modes, start using libraries, etc. The setup function will only run once, after each powerup or reset of the Arduino board. After creating a setup() function, the loop() function loops consecutively, allowing the program to change and respond.

Step 4 - Integrate Arduino compiler

arduino.mk

Generate Makefile with Acceleo

make 


BOARD_TAG    = uno
ARDUINO_PORT = /dev/ttyACM0
ARDUINO_LIBS = Servo

include /usr/share/arduino/Arduino.mk
When you use the Arduino environment the compile and upload actions are directly integrated. Here as I want to integrate the generation and the compilation of ino files in my Eclipse application, I searched how to compile and upload Arduino programs from the command line. As I am a Ubuntu user, I just grab the arduino-mk package and set up some environment variables. Then I updated my generator in order to generate also small Makefile for the sketch. Then the upload action will just have to call the make command for building. After building, a new directory has been created which contains all the object files.

Arduino designer development

Step 5 - Integrate target uploader

avr-gcc, avrdude

make upload

To be able to upload the code on the target, I also installed avr-gcc which is a cross-compiler for the AVR chip that runs the Arduino and avrdude. avrdude is a very handy utility for loading code onto Atmel AVR microcontrollers. As Arduino ships with its own bootloader, it is possible to load and run code using only a single USB cable connecting the computer to the board. This cable comes with my Ardublock kit. Thanks to arduino.mk, uploading the code on the hardware platform is then really easy. My Eclipse upload action has just to call the make upload command.
